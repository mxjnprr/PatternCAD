# Story 002-01: Create and manage parameters

**Epic:** EPIC-002 - Parametric Design and Constraint System
**Status:** Not Started
**Estimate:** 5 points
**Assigned To:** Unassigned

---

## Description

As a pattern designer, I want to create named parameters with values and units
so that I can define dimensions that can be reused throughout my pattern.

---

## Acceptance Criteria

- [ ] User can create new parameter via Parameters Panel
- [ ] Each parameter has name, value, unit (mm, cm, inches)
- [ ] Parameter names must be unique and valid identifiers (alphanumeric + underscore)
- [ ] User can edit parameter values inline
- [ ] User can delete parameters (with warning if referenced by geometry or other parameters)
- [ ] Parameters can be organized into groups/categories (e.g., "Measurements", "Allowances")
- [ ] Parameters Panel shows all defined parameters in table format

---

## Technical Notes

**Parameter Class:**
```cpp
class Parameter {
    QString name;           // e.g., "waist_width"
    double value;           // numeric value
    Unit unit;              // mm, cm, inches
    QString expression;     // formula (empty if direct value)
    QString group;          // category/group name
    std::vector<Parameter*> dependencies;  // params this depends on
};
```

**ParametersPanel Implementation:**
- Use QTableWidget or QTableView with custom model
- Columns: Name, Value, Unit, Expression, Group
- Enable inline editing with validators
- Implement Add/Delete buttons
- Validate name uniqueness on input
- Show warning dialog if deleting referenced parameter

**Validation:**
- Name: alphanumeric + underscore, no spaces, must start with letter
- Value: numeric, must be valid for unit
- Unit: enum {mm, cm, inches}
